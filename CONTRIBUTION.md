## How Can I Contribute?

### Reporting Bugs

When reporting bugs, please include:

* A clear and descriptive title
* Steps to reproduce the issue
* Expected vs actual behavior
* Environment details (OS, language/runtime version, etc.)
* Logs, screenshots, or error messages if applicable

---

### Suggesting Enhancements

Enhancement suggestions are welcome! Please provide:

* A clear description of the proposed change
* Motivation and use cases
* Possible alternatives or drawbacks

---

### Submitting Code Changes

Before submitting code changes:

* Check existing issues and pull requests
* Discuss major changes via an issue first
* Ensure your changes align with project goals

## Commit Message Guidelines

Follow a consistent commit message format.

Example:

```
<type>(<scope>): <short summary>

[optional body]
```

Types may include: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`

## Pull Request Process

1. Create a feature or bugfix branch
2. Make your changes
3. Add or update tests as needed
4. Update documentation if applicable
5. Submit a pull request with a clear description

Pull requests should:

* Reference related issues
* Pass all CI checks
* Be reviewed and approved before merging
